의류 이미지 검색 엔진 개발
목적
딥 러닝 기반 의류 검색 엔진 개발 
환경
Ubuntu14.04,  Python,  Theano,  Keras 
내용
기존의 검색 방법은 의류에 태깅하여 분류하는 방법을 이용. 본 엔진에 사용된 방법은 이미지 자체를 학습하여 사용자가 원하는 옷과 비슷한 색상, 디자인의 옷을 검색해주는 엔진. 크롤링하여 다양한 웹 사이트의 옷 이미지와 URL을 수집하여 DB를 구성하고 이미지의 특징 벡터를 생성. 특징 벡터 생성 시 색상과 디자인을 따로 분류하여 생성하고 따로 학습을 진행. 해외 의류는 약 90% 이상의 정확도를 보였으나 국내 의류는 75% 정도의 정확도를 보임. 국내 의류는 배경도 많고 수집 시 분류되는 디자인의 이름이 쇼핑몰 마다 다르게 표기 되어 같은 디자인의 의류라도 다르게 불리고 있는 단점이 있음. 초기 DB 생성 시 잘못 분류되어 벡터가 잘못 생성되는 것이 많은 오류를 가지고 있는 것으로 파악되었고 국내 의류는 태깅을 함께 이용하여 고도화를 진행.

ImageNet 2014에 사용된 VGGnet를 기본으로 실시간 운전자 상태 판단 시스템을 개발. 운전자의 시선 위치를 총 8곳으로 나누어 데이터셋을 구성하였고 이를 학습시켜 실시간으로 시선을 판단하게 개발. 사람의 머리 포즈(3D-Yaw, Pitch, Rall)를 단일 카메라로 판단 가능. 사람의 머리에 센서를 부착하여 카메라로 촬영된 머리 이미지와 3축의 각도를 동시에 얻어 트레이닝 데이터셋으로 구성. 약 1.5도 오차. 얼굴 특징점을 얻을 수 있어 눈 깜빡임과 하품 등의 상태를 판단 가능. 운전자 상태 판단을 모두 CNN을 이용하여 구성. 단일 카메라로 하품, 졸음 (머리의 각도), 시선(시선 부주의) 등을 판단. 기존에는 다양한 센서를 이용했어야 했지만 단일 카메라로 구성할 수 있다는 것이 가장 큰 장점.

DBASM(Discriminative Bayesian Active Shape Model) 
목적
기존 ASM + MOSSE FILTER를 적용한 얼굴 모델 
환경
Windows,  C++,  OpenCV 
내용
기존 ASM 얼굴 모델의 각 특징점에 MOSSE Filter를 적용하여 특징점 자체를 추적하여 만드는 얼굴 모델. 특징점의 MOSSE Filter가 업데이트 되어 폐색 상황에서도 지속적인 얼굴 모델 제작 가능. 얼굴 모델이 추적할 수 있기 때문에 POSE Estimation도 가능. 부분적인 움직임에도 얼굴 모델 제작이 가능하기 때문에 전남대와 연계하여 통풍 환자의 얼굴을 보고 병의 유무를 판단할 수 있는 프로그램을 개발. 
<전남대학교 시범테스트 실행화면>
MOSSE Filter(Minimum Output Sum of Squared Error) 
목적
변화에 강인한 객체 추적 알고리즘 
환경
Windows,  Matlab,  C++ 
내용
FFT를 이용하여 이미지를 주파수 도메인으로 변경하여 필터 제작. 필터를 이용하여 현재 위치에서 가장 비슷한 곳으로 필터가 이동. 폐색, 회전, 조명변화 등으로 찾으려는 객체의 변화에도 주파수 도메인으로 변경된 필터에는 큰 영향을 미치지 않아 지속적인 추적 가능.
AAM(Active Appearance Model) 
목적
차량 내부에서 단일 카메라를 이용한 얼굴 특징점의 변화 검출 
환경
Windows,  C++,  OpenCV 
내용
차량 내부에서 운전자의 상태를 파악하여 졸음운전, 전방 부주의 등을 판단하기 위한 연구를 진행. 운전자의 상태 파악을 위해 눈의 변화, 입의 변화, 얼굴 각도 등이 필요. 여기에 적용하기 위해 AAM 모델을 사용하였고 얼굴 특징점 58개를 학습. 다양한 얼굴 사진을 구하고 58개의 얼굴 특징점을 생성 후 학습, PCA를 이용하여 평균 Shape를 계산. 일반적인 상황에서 얼굴  특징점 추출이 가능해짐.
Adaptive Template Matching(적응형 템플릿 매칭) 
목적
폐색에 강인한 객체 추적 시스템
환경
Windows,  C++,  OpenCV 
내용
기본적인 템플릿 매칭을 이용한 객체 추적시스템은 객체의 변화에 매우 취약함. 이를 극복하기 위해 템플릿을 주기적으로 업데이트를 시키는 방법을 고안. 업데이트의 조건으로 Outlier 개념를 추가하여 Outlier의 수치가 일정 수준 이상 발생하면 템플릿을 업데이트 시킴. 추가적으로 Kalman Filter를 이용하여 이전 템플릿과의 업데이트 되는 템플릿의 차이를 줄여 급격한 템플릿 변화를 방지. 폐색의 면적을 20%, 40%, 60%로 증가하면서 실험을 진행. 기존 템플릿 매칭 은 약 75%, 50%, 15%의 정확도를 보였으며 적응형 템플릿 매칭은 약 90%, 85%, 78%의 정확도를 보여 향상시킴. 템플릿 매칭의 계산속도로 인하여 실시간 적용이 어려움.
